# LGPL compliance and replacement instructions

Nodus uses unmodified LGPL libraries while keeping the Nodus application code
under MIT. The LGPL components are unpacked from Electron's `app.asar`, so they
remain separately replaceable. Nodus permits reverse engineering when needed
to debug modifications to those libraries. This distribution uses the suitable
shared-library mechanism described by LGPLv3 section 4(d)(1): the application
loads these components at runtime and can operate with ABI-compatible modified
versions installed in their place.

## Exact corresponding sources

### HEIC decoder

- npm package: `libheif-js` 1.19.8
- package source: https://github.com/catdad-experiments/libheif-js/tree/fe8e9c29440b839910be9dc32e8d2b826c8217ca
- embedded libheif source: https://github.com/strukturag/libheif/tree/v1.19.8
- license: LGPL-3.0

### Sharp/libvips native bundle

- npm packages: `@img/sharp-libvips-*` 1.2.4
- exact packaging/build source: https://github.com/lovell/sharp-libvips/tree/20b5e899954907a3039d6e3d4c200aaa0ec52c4c
- libvips source: https://github.com/libvips/libvips/tree/v8.17.3
- complete embedded-library version list and upstream notice:
  `legal/generated/SHARP_LIBVIPS_VERSIONS.properties` and
  `legal/generated/SHARP_LIBVIPS_THIRD_PARTY_NOTICES.md`
- LGPL libraries are used under their LGPLv2/v2.1 "or later" clauses as LGPLv3.

Nodus does not modify these libraries. The source of the corresponding Nodus
application code is https://github.com/Drakonis96/nodus and each release also
has a source archive generated by GitHub. The exact npm dependency graph is in
`package-lock.json`. If any referenced upstream source or replacement path
becomes unavailable, report it at https://github.com/Drakonis96/nodus/issues.

## Rebuild with a modified library

The most reliable replacement path is to rebuild the open-source application:

1. Check out the Nodus source corresponding to the installed release.
2. Install the exact dependency graph with `npm ci`.
3. Replace/build `node_modules/libheif-js` or the relevant
   `node_modules/@img/sharp-*` and `node_modules/@img/sharp-libvips-*` packages.
4. Run `npm run dist:mac`, `npm run dist:win` or `npm run dist:linux`.

The Nodus TypeScript source and build scripts are the Corresponding Application
Code needed to produce a combined work with a modified library.

## Replace a library in an installed application

The relevant files are deliberately emitted under
`resources/app.asar.unpacked/node_modules/` (under `Contents/Resources` inside a
macOS `.app`). Replace the matching package files there with ABI-compatible
builds. On macOS, changing a signed bundle invalidates its signature; after
making a personal modified copy, it can be ad-hoc signed again with:

```sh
codesign --force --deep --sign - /path/to/Nodus.app
```

Operating-system security may require explicitly allowing that personal build.
Nodus provides no warranty for modified libraries, but does not prohibit their
installation or debugging.
